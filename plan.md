￼# **LLM Self‑Critique / Identity Bias Research Proposal**

## **1. Background and Motivation**

Recent research (2024–2026) shows several consistent findings about large language models (LLMs):

1. **LLMs struggle to correct their own mistakes.** Many studies highlight a **self‑correction blind spot** where models fail to fix errors in their own outputs, even when they correct the same errors if those outputs are attributed to another source.
2. **Self‑preference and self‑attribution bias.** LLMs often rate their own answers more favorably than equivalent answers credited to other models, and this bias persists across various tasks.
3. **Context anchoring.** Reviewing within the same conversation context leads models to anchor on their initial output and rationalize, rather than scrutinize, their answers.
4. **Cross‑model review is stronger.** External or cross‑model critiques are more reliable than self‑critique. Multi‑agent and role‑based methods improve performance but increase cost and complexity.
5. **Source framing affects evaluation.** Changing the stated author (human, other model, specific nationality) can significantly shift evaluation scores, even when the content remains the same.

These findings suggest that the core problem is not strictly reasoning capability but **perspective, attribution, and context bias** during critique.  This proposal explores how attribution and context influence error detection and correction in reasoning tasks and whether removing such biases leads to better outcomes.

---

## **2. Research Questions**

1. **RQ1:** Do LLMs detect errors differently depending on **who they believe wrote the solution**, even when the solution itself is identical?
2. **RQ2:** Is self‑correction failure primarily caused by **self‑attribution bias**, **context anchoring**, **identity bias**, or inherent reasoning difficulty?
3. **RQ3:** When we remove attribution or context bias, **does reasoning accuracy improve** in practice?

---

## **3. Main Experiment: Identity Swap Critique**

### **Concept**

Use the **same solution** across all conditions but change only the **authorship label** presented to the critic.  This allows us to measure the pure effect of attribution on error detection and correction.

### **Conditions**

For each identical solution, present it to the critic with one of the following labels:

1. **Self** – “This is your previous solution.”
2. **Other model** – “This solution was generated by GPT‑4.”
3. **Weak model** – “This solution was generated by a small model.”
4. **Anonymous** – “Here is a solution.”
5. **Human** – “A student wrote this solution.”

### **Critic Tasks**

For each condition, ask the critic model to:

- Determine whether the solution contains an error.
- Identify the location of the error.
- Provide a corrected version.
- Estimate confidence in its critique.

### **Metrics**

- **Error detection accuracy**
- **Error localization accuracy**
- **Correction accuracy**
- **False positive rate**
- **Confidence calibration**
- **Performance gap between conditions**

### **Datasets**

Select reasoning datasets that require non‑trivial chain‑of‑thought solutions, for example:

- GSM8K (grade‑school math)
- MATH (college‑level proofs)
- AIME/AMC problems
- GPQA (general practice questions)
- Logic puzzles
- Code reasoning tasks

This experiment directly measures the impact of **authorship framing** on critique performance.

## **4. Context Separation Experiment**

### **Purpose**

This experiment distinguishes between **context anchoring** and **identity bias**.  It examines whether simply changing the session context (without altering the authorship label) improves critique performance.

### **Conditions**

| **Condition** | **Description** |
| --- | --- |
| Same session | Review occurs in the same conversation context |
| New session | Review happens in a new session |
| New session + anonymous | New session without authorship information |
| New session + paraphrased | New session with paraphrased solution |
| Cross‑model critic | A different model performs the review |

### **Interpretation**

- **Same session** → measures baseline and anchoring effect.
- **New session** → isolates self‑attribution by removing conversation history.
- **Anonymous** → removes identity cues while keeping context separate.
- **Paraphrase** → tests surface‑pattern bias.
- **Cross‑model** → compares self‑critique vs. external critique.

Comparing these conditions helps decompose sources of bias.

---

## **5. Mitigation Experiment**

### **Goal**

Investigate if and how removing bias factors leads to concrete improvements in reasoning accuracy and error correction.

### **Potential Mitigations**

- **Anonymous solutions** – remove identity cues altogether.
- **Paraphrased solutions** – rephrase the chain of thought to reduce surface pattern recognition.
- **New session review** – separate production and review contexts.
- **Debate or multi‑agent critique** – use multiple agents with differing roles.
- **Verifier model** – use a secondary model dedicated to verification.
- **Tool‑based verification** – integrate external tools (e.g., compilers or theorem provers) to check parts of the output.
- **Confidence‑guided review** – adjust critique based on confidence thresholds.

### **Evaluation**

For each mitigation, compare:

- Detection accuracy
- Correction accuracy
- Final task accuracy
- Confidence calibration and harmful corrections

The aim is to identify simple protocols that reduce bias and improve performance.

## **6. Paper Storyline**

A clear narrative strengthens the paper:

1. LLM self‑correction is widely considered weak.
2. Existing work often treats this as a **reasoning limitation**.
3. We hypothesize the failure arises from **perspective and attribution bias**.
4. The **Identity Swap** experiment shows the same solution is judged differently based on its attributed author.
5. The **Context Separation** experiment reveals anchoring and context length effects independent of identity.
6. **Mitigation experiments** demonstrate that removing identity/context cues leads to better error detection and correction.
7. We conclude that self‑correction failures are largely due to evaluation protocol design, not fundamental reasoning ability.

This storyline positions the research as both an evaluation‑bias study and a practical contribution for improving self‑correction protocols.

## **7. Experimental Pipeline**

A typical pipeline for these experiments:

```jsx
Problem → Solver Model → Solution
Solution → Critic Model → Error Detection
Solution → Critic Model → Correction
Evaluate Detection / Correction / Accuracy
```

**Identity Swap Version**

```jsx
Solution
  ↓
Add label (self / other / anonymous / human)
  ↓
Critic
  ↓
Detection / Correction / Confidence
```

This modular design allows easy swapping of labels and contexts while keeping the solver and tasks fixed.

## **8. Analysis Strategies**

Important analyses include:

- Size of the **self vs other critique gap**
- Relationship between **task difficulty** and bias
- Impact of **model size** on bias
- Correlation between **detection and correction success**
- Calibration of confidence across conditions
- **False correction rate** differences
- Effects of **context length** (short vs long solutions)
- Impact of **paraphrasing** and anonymization on detection accuracy

Detailed analyses will clarify where bias arises and how it can be mitigated.

## **9. Research Roadmap**

### **Step 1 — Literature Review**

Study prior work on self‑correction, self‑preference bias, critic training, multi‑agent debate, Cross‑Context Review, and source‑framing bias.

### **Step 2 — Dataset Selection**

Curate a mix of math, logic, and code reasoning tasks (e.g., GSM8K, MATH, AIME, GPQA).

### **Step 3 — Build Pipeline**

Develop scripts to generate solutions from a solver model, apply identity labels, and run critics under specified conditions.

### **Step 4 — Identity Swap Experiment**

Implement the primary experiment across datasets and collect metrics.

### **Step 5 — Context Separation Experiment**

Run reviews in both same-session and new-session contexts, with and without identity cues.

### **Step 6 — Mitigation Experiment**

Test anonymization, paraphrasing, and multi‑agent methods to reduce bias.

### **Step 7 — Analysis and Visualization**

Aggregate results, perform statistical tests, and visualize performance differences.

### **Step 8 — Write Paper**

Organize findings into a coherent narrative highlighting protocol design as the key to better self‑correction.

## **10. Expected Contributions**

- **Evidence** that identity and context framing substantially influence error detection and correction in LLMs.
- **Bias decomposition** showing the relative impact of self‑attribution, anchoring, and identity labeling on critique performance.
- **Simple mitigation protocols** (new session review, anonymization, paraphrasing) that improve self‑correction.
- **Guidelines** for designing evaluation and debugging procedures for LLM reasoning tasks.

If successful, these contributions could be publishable in top NLP/ML venues and provide practical insights for building more reliable LLM systems.

## **11. Final Remarks**

This proposal frames self‑critique failures as arising from evaluation protocol design rather than inherent reasoning limits.  By systematically varying authorship labels and context, we can diagnose biases and propose effective, low‑cost mitigations.  The experiments and analyses outlined here form a complete research project with the potential for significant impact.
